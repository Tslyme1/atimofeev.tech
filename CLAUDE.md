# Главная сайт — конвенции проекта

Многостраничный статический сайт (портфолио + кейсы). Общая цель: **максимальное переиспользование**
блоков между страницами. Перед добавлением нового UI — сначала ищи, есть ли уже подходящий класс/компонент
ниже, и используй его вместо нового markup.

## Структура файлов

- `index.html` — главная/портфолио: сайдбар-профиль, hero, карточки кейсов.
- `design-system.html`, `studio.html`, `monetization.html`, `video.html` — 4 кейс-страницы, у каждой свой
  `.subnav`/TOC и якорные `#section`.
- `styles.css` — один файл на 1180+ строк, разбит на секции `====` баннер-комментариями по компонентам.
  Не дробить на несколько файлов без запроса пользователя.
- `script.js` — один файл на все страницы (`<script src="script.js">`), секции `// ---- comment ----`:
  nav drawer, case modal, profile modal, клик-навигация по карточкам, RU/EN i18n словарь + TreeWalker.

## Design tokens (`:root`, styles.css:4-35) — ВСЕГДА использовать вместо хардкода

```css
--bg #f6f6f6   --surface #fff   --surface-2 #eeeeee
--ink #24262b  --muted #9a9a9a
--dark #17181a --dark-2 #1d1d1e
--accent-pink #eb2e9f  --accent-blue #4f7dff  --accent-yellow #fec901  --accent-green #22c57a

--radius-xl 32px  --radius-lg 24px  --radius-md 20px  --radius-sm 12px  --radius-pill 999px
--gap 8px  --pad 32px
--sidebar-w 398px (→240px при ≤1300px)  --fab-size 64px  --side-margin 48px  --content-max 1418px
--ff-display / --ff-body: 'Inter', system-ui, sans-serif
```

Известные повторяющиеся "сырые" значения, которые встречаются в коде без токена — **при следующей правке
переводить на переменную, а не копировать дальше**:
- `#0c0c0e` (тёмная панель) — использовать `--dark` или завести `--dark-3`, если оттенок реально другой.
- `#be32ff` / `rgba(190,50,255,*)` — "violet" акцент, встречается 5+ раз как сырой цвет — стоит завести `--accent-violet`.
- `#d9d9d9` (дашед-плейсхолдер), `#ececec` (граница footer-nav, близко к `--surface-2` но не равно) — повторяются буквально, надо на переменные.
- `border-radius:24px` пишется буквально в нескольких местах вместо `var(--radius-lg)` — исправлять по ходу правок.

## Переиспользуемые компоненты (обновлено)

- **`.media-duo`** — заголовок+подзаголовок+2 картинки (studio.html, блок "О мобильной студии" и др.).
  Отступ `margin-top:24px` уже в CSS, инлайн-стили не нужны.
- **`.info-row` / `.info-row.plain`** — инфо-блок "Роль/Что сделано/Аудитория". Унифицировано: везде
  (studio/monetization/video) используется модификатор `.plain` (прозрачный фон). Отступ сверху (24px)
  теперь в CSS по умолчанию, в HTML инлайн-стили убраны.
- **`.card-slider`** — общий класс-миксин для горизонтального слайдера карточек (scroll-snap).
  Добавляется вторым классом к контейнеру: `class="flow-cards card-slider"`, `class="funnel-row card-slider"`,
  `class="compare-panels card-slider"`. Сам слайдер не диктует стиль карточек внутри — оформление карточки
  (`.flow-card`, `.funnel-step`, `.panel`) остаётся своим. При добавлении новой секции
  "заголовок + слайдер карточек" — добавляй `card-slider` к контейнеру, не изобретай новый scroll-механизм.
- **`.compare-panels`** — пара "было/стало" (design-system.html): `.compare-panels > .panel(.dark)` с
  `h3`+`p`+одна-две `img` (текст сверху, картинка(и) по центру ниже). Добавляй `card-slider` вторым
  классом на контейнер, чтобы пара скроллилась на планшете/мобилке.
- **`.stat-row.tiles`** — плашки метрик (studio/video "Результаты"). По умолчанию белые на светлом фоне;
  модификатор `.dark` (`.stat-row.tiles.dark`) — тёмные плашки на тёмном тексте (design-system.html).
  Растянуты на всю ширину, gap 8px.

## Известные несогласованности — не копировать дальше, чинить при следующей правке в этом месте

1. **Sidebar-профиль продублирован 4 раза** (index sidebar, index `.profile-modal__panel`, studio
   `.case-modal__profile`, monetization `.case-modal__profile`) — идентичный markup вставлен копипастой.
   При правке контента профиля (имя, опыт) — искать и обновлять во всех 4 местах, либо предложить
   пользователю вынести в шаблон/JS-инклюд.
2. **Два разных паттерна гамбургер-меню** используются вперемешку: design-system/video → простой
   `.nav-drawer`/`.scrim`; studio/monetization → богатый `.case-modal` (с TOC и профилем). Один и тот же
   `#menuFab` ведёт к разным UI в зависимости от страницы. Не добавлять третий вариант — выбрать один
   паттерн, если просят унифицировать.
3. **Кнопка "назад" в `.subnav`** — на design-system/monetization/video это `javascript:history.back()`
   с текстом "Назад"; на studio.html — жёсткая ссылка на `index.html` с текстом "Главная". При правке
   navigation на любой странице — использовать вариант `history.back()`/"Назад" как основной.
4. **`.footer-nav` отсутствует на studio.html** — там вместо этого только `.teaser-row`. Если унифицируем
   концовки страниц — добавлять `.footer-nav` по образцу остальных 3 страниц.
5. **Текст pill в `.teaser-card`** различается без причины: index/video — "Case study",
   studio/monetization — "Читать кейс". Использовать один вариант при новых тизерах (уточнить у пользователя, какой).
7. **Hero задаёт `min-height` инлайн-стилем** по-разному на каждой странице (420px/0/450px) — вместо этого
   заводить модификатор `.hero--compact` с токеном, а не новое magic-number.
8. **`design-system.html` — единственная страница**, где feature-иконки это эмодзи в тексте, а не `<img>`
   как на index/studio — не плодить третий способ рендера иконок фич.

## Инлайн-стили — избегать, выносить в классы

В HTML много `style="margin-top:16px"` / `24px` / `8px` разбросано по всем кейс-страницам вместо
переиспользуемых utility-классов или существующих токенов `--gap`/`--pad`. При правке рядом стоящего
блока — заменять инлайн на класс, а не добавлять ещё один инлайн-стиль.
Пример: `monetization.html` 4 раза повторяет один и тот же длинный `style="color:var(--muted);
font-size:18px; line-height:1.5; margin:0 0 24px; padding-left:22px; max-width:700px"` на списках —
это кандидат на класс `.lede-list`.

## Naming convention

BEM-ish, не строго: блок kebab-case (`.profile-card`), элемент через `__` (`.profile-card__avatar`),
модификатор — просто доп. класс без `--` (`.showcase.fill`, `.card.plain`, `.icon-chip.violet`), а не
`.block--modifier`. Придерживаться этого стиля для новых компонентов. ID — только для JS-хуков,
camelCase (`#menuFab`, `#caseModal`).
